# ToneSolo 移动端开发计划 (Android)

基于 Monorepo 架构和现有的核心共享模块 (`packages/core`)，本计划旨在高效构建 Android 端应用。

## 1. 开发目标与功能需求

**核心目标**：构建一个功能完备、体验流畅的 Android 音乐播放器，实现与 PC 端的高度互通和代码共享。

**功能需求**：
*   **基础功能**：账号登录/退出、歌单管理、音乐播放（在线/本地）、歌词显示、搜索。
*   **移动端特性**：后台播放控制、锁屏控制 (MediaSession)、沉浸式状态栏、触控手势交互。
*   **核心模块复用**：复用 Core 包中的 API 请求、Pinia 状态管理、工具函数。

## 4. 技术实现方案

*   **架构模式**：Monorepo (pnpm workspace)
*   **核心框架**：Vue 3 + TypeScript + Vite
*   **状态管理**：Pinia (复用 `@tonesolo/core`)
*   **UI 框架**：**Vant 4** (轻量级移动端 Vue 组件库) + 自定义 CSS
*   **跨平台容器**：Capacitor 6 (提供 Android Runtime)
*   **路由管理**：Vue Router (Mobile 独立路由配置)
*   **图标库**：复用 `packages/desktop` 中的 SVG 或引入 Iconify

## 5. 质量保证与测试

*   **单元测试**：复用 Core 包已有的逻辑测试。
*   **真机调试**：使用 Android Studio + 实体机/模拟器进行调试。
*   **UI 走查**：确保不同尺寸屏幕 (手机/平板) 的适配性。
*   **性能测试**：关注内存占用和耗电量，确保后台播放不被杀后台。

## 6. 资源与风险

*   **风险点**：
    *   **Android 后台保活**：Android 系统倾向于杀后台进程，需配置 Capacitor 插件及前台服务。
    *   **样式兼容性**：Desktop 的 CSS 变量可能不完全适配移动端，需建立移动端的主题变量层。
*   **应对**：提前引入 `capacitor-background-mode` 或类似插件；建立 `mobile/src/styles/vars.css` 覆盖 Core 中的变量。
